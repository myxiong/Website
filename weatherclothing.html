TUSK

Clothing Weather Mobile Application

Tusk is a mobile application that notifies the users to bring their essentials with them depending on the weather conditions. This project was developed by myself and a friend. We found that one of the main reasons why people check the weather is to plan for what to prepare for the day. We wanted to improve this preparation process by combining the two actions into one.

Personas

There are three main users we are targeting:

  1. Rick 36.
    Busy father of 3. Constantly on the move with his kids. He has the tendency to forget essentials (i.e. sunscreen, hat, umbrella) for him and his children. He lives in California, typically very warm and sunny place. He is looking for something that will help remind him what he needs to bring that day for him and his kids.

  2. Sara 24.
    New graduate, working in the heart of Toronto. She likes to walk to and from work. Torontoâ€™s temperature can range from sunny to snowy just within a few days. Clothing articles can change from day to day. She is looking for something that will help her dress accordingly for her daily commute.

  3. Joe 27.
    Travels very often between cities to cities for work. He typically stays in these locations for a week or two. Joe is always on the move hence he needs to know what to pack beforehand for his trip.

Workflow

User lands on login page. By default, login page leads user to the main weather screen where all the important details are disabled as explained in the next section. User can go to menu to access all the other pages. Menu can include weather screen, inventory screen, location screen, and profile screen.

Location screen contains all the locations and its respective weather the user has added. User can click on each location to see its weather and suggested items.

Inventory screen disables all the items user can use. From inventory screen, user can access each item and customization screens as explained in the following section.

Weather Screens

There are four main sections to the weather screen: daily section, hourly, weekly, and other items.

Daily section: contains details on current weather and items required. These items are the important ones that user configured for notification. Also includes time, date, location information.

Hourly weather: graphical representation of the upcoming weather information. In addition, it shows what essential items should be brought out for hourly duration. If user wants to see a specific time in more detail, can click to update daily section. Time and date information will indicate what time period is currently selected

Weekly weather: displays weather details for the upcoming days.

More items: These items are what user should consider bringing but not essential to their trip. These items will only be displayed if user opens application and will not notify user.

Adding Conditions

A big feature of this application is giving users the power to configure when items will appear for the user.

  1. Details of item is viewed. Includes weather conditions and the status of item (i.e. set to enable or notify). Underneath conditions is option to add more conditions.

  2. To add condition, user must select specific weather conditions before adjusting the weather details.

  3. User can search for weather conditions. Weather conditions that are not similar in occurrence will be disabled.

  4. User can now adjust individual weather condition details. There are 7 weather details available.

  5. For each condition, user has a scale to indicate where they like the item to fall under. Depending on what weather condition as selected, some parts of the scale will be disabled (error checking).

Icons

There were two main sets of icons created for this project: items icons and weather icons. Item icons are categorized based on the type. The item icons will eventually cover almost all of the most commonly used item not limited to location. Icons can be in 3 states: disabled, enabled, or selected. The states of these icons are determined either when user manually selects the state or when that item/weather is restricted due to conditions.
